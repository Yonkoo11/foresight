## QUESTION 1: What's the Core Loop?

**Model:** Nir Eyal's Hook Model (4-step engagement cycle)

```
TRIGGER (Competitive Anxiety)
  "Friend beat me #45 vs #47"
                ↓
ACTION (Minimal Friction)
  "Check scores" — 1 tap, 10 seconds
                ↓
VARIABLE REWARD (Unpredictable)
  "My rank changed!" — updates every 30 seconds
  Uncertainty = addiction (like slot machines)
                ↓
INVESTMENT (Social Commitment)
  "Shared team on X, friends are watching"
                ↓
LOOP REPEATS
```

**Why It's Addictive:**
- Live scoring every 30 sec = 1,440 potential dopamine hits per day
- Leaderboard proximity (friends nearby) = local competition
- Variable reward timing (not predictable when score will move) = increases dopamine
- Social visibility (team shared on X) = prevents quitting

**Comparison to Competitors:**
- DraftKings: Weekly loop (scores update 1x/week) = lower engagement
- **Foresight: 30-second loop = 168x more engagement opportunities**

## QUESTION 2: Where's the "Aha Moment"?

**Definition:** The second a user understands the core mechanic and wants to keep playing.

**For Foresight: "My captain just got 45K likes, and I earned +12 points in real-time."**

### Exact Timeline

| Time | Event | User State |
|------|-------|-----------|
| T+0:00 | Click "Join Contest" | Curious |
| T+1:00 | Draft page loads | Engaged (see influencer options) |
| T+2:30 | Pick captain, see budget | Learning (understand constraints) |
| T+3:00 | Submit team | Committed (team locked) |
| T+3:30 | Leaderboard shows rank #47 | Competitive (now ranked vs others) |
| T+4:00 | Scores start updating live | **Aha Moment** |
| T+4:15 | See own rank move from #47→#45 | **Hooked** ✅ |

**Aha Success Rate Hypothesis:** If user reaches live scoring by T+4:15, they have 60%+ probability of returning Day 2.

**Red Flag:** If Aha occurs after T+5:00, or if <50% who see aha return Day 2, redesign flow.

### Why This Aha Works

1. **Proof of Skill:** User sees real-time results tied to their picks
2. **Competitive Clarity:** Rank is unambiguous (am I winning or losing?)
3. **Progress Visibility:** Score breakdown explains why they earned points
4. **Social Proof:** Leaderboard shows they're competing vs. real people

|---------|---|---|---|---|
| **Leaderboard Proximity** | +15-20% | Low (filter + sort) | ⭐⭐⭐⭐⭐ | **PRIORITY** |
| **Rank Change Animation** | +20-30% session | Low (CSS animation) | ⭐⭐⭐⭐⭐ | **PRIORITY** |
| **Team Replicability** | +10-15% | Medium (UI + backend) | ⭐⭐⭐⭐ | Include |
| **Activity Feed** | +8-12% | Medium (backend done) | ⭐⭐⭐ | Phase 2 |

| **Likes** | +0-2% | Low | ❌ | Skip |
| **Comments** | +2-5% | High (moderation) | ❌ | Skip |

### Why "Vanity Features" Don't Drive Growth

- **Comments:** Users don't read comments, prefer DMs on X
- **Likes:** Doesn't drive behavior change, just feels good
- **Follows:** Users don't care about following strangers, prefer friends

### What Actually Drives Growth

**Leaderboard Proximity** (Make It Default)
- Users don't want a global leaderboard they can never win
- Users OBSESS over beating friends ranked 3 spots away
- Show top 5, user rank, bottom 5 → hide the impossible-to-win middle
- Retention lift: +15-20% (strongest single lever)

**Mechanism:** User's friend is #45, they're #47
→ Refresh 20x to see if they overtook them
→ Daily habit forms around this rivalry

**Rank Change Animations** (Visual Feedback)
- When scores update, show ↑/↓ with change magnitude
- "↑ 5 spots" in green = celebration dopamine
- "↓ 2 spots" in red = urgency to fix
- Retention lift: +20-30% on session engagement
- Cost: 2-3 hours, high ROI

**Team Replicability** (Learning Mechanism)
- Top-10 teams visible on leaderboard
- "Copy this team" button → fills your roster with theirs
- Users learn winning strategies
- New players get unblocked (don't need to understand tiers)
- Retention lift: +10-15%

## QUESTION 6: Leaderboard as a Growth Engine

### Current State: "People ranked by score" (Boring)

### Optimized State: "Local competition that creates obsession"

**Design Principles:**

| Principle | Implementation | Behavioral Effect |
|-----------|-----------------|-------------------|
| **Proximity** | Show top 5, user rank, bottom 5 (not all 500) | Users focus on beatable competitors |
| **Status Animation** | ↑ green, ↓ red, animated rank changes | FOMO + celebration |
| **Comparison** | "You: #47, Friend A: #45, Friend B: #50" | Local rivalry = daily engagement |
| **Volatility** | "You were #52 yesterday" | Momentum tracking |
| **Badges** | Gold ⭐ for top-10, silver for top-50 | Status symbols |
| **Trend Streaks** | "↑ 5 spots this week" | Momentum |
| **Milestones** | "You hit top-100! 🎉" Toast | Celebration moments |

### Leaderboard-Driven Loop

```
User sees friend is #45 (they're #47)
    ↓ (TRIGGER: Status anxiety — "I'm losing")
User opens app to check scores
    ↓ (ACTION: 1 tap)
Scores updated, user now #46 (or #48)
    ↓ (VARIABLE REWARD: Uncertainty)
User shares rank on X, forms rivalry with friend
    ↓ (INVESTMENT: Public commitment)
Returns tomorrow to beat friend
    ↓ (RETENTION: Sustained)
```

### Must-Build Leaderboard Features

**Priority 1 (High Impact, Low Effort):**
1. **Friend Comparison View** (4-6 hours)
   - "Friends" tab on leaderboard
   - Show proximity (3 above, 3 below)
   - Retention lift: +10-15%

2. **Rank Change Animations** (2-3 hours)
   - Animate rank movement (↑/↓) when scores update
   - Color: green for up, red for down
   - Engagement lift: +20-30% session length

3. **Milestone Notifications** (1-2 hours)
   - Toast: "You hit top-100!" "You beat your friend!"
   - Celebration moments = retention boost
   - Lift: +5-8%

**Priority 2 (Medium Impact):**
4. **Trending Teams** (4-5 hours)
   - Show fastest-rising teams (highlight edge picks)
   - Lift: +5-8% (inspiration)

5. **Head-to-Head Record** (4-5 hours)
   - "vs @friend_name: You: 3-1"
   - Rivalry tracking
   - Lift: +10-15%

## QUESTION 7: What Kills Growth? (Anti-Patterns)

### #1: Auth Friction (60-70% bounce)
**Failure:** User sees Privy modal, thinks "wallet thing," closes, bounces
**Fix:** Landing shows "Email or Social Login (No Wallet)" + social buttons first

### #2: Slow Live Scoring (30-40% active churn)
**Failure:** User opens leaderboard, scores haven't updated in 3 hours
**Fix:** SSE every 30 sec, polling fallback, status badge shows "🟢 Live"

### #3: Unclear Scoring (20-30% retention loss)
**Failure:** Captain got 45K engagement, user earned +8 pts — why not +12?
**Fix:** Hover tooltip: "Viral threshold 50K, you had 45K, +1 pt (instead of +4)"

### #4: No Social FOMO (50-60% lower retention)
**Failure:** Leaderboard is strangers, no friends to beat
**Fix:** Seed demo accounts, influencer seeding, friend import from Twitter

### #5: No Influencer Incentives (10x smaller growth)
**Failure:** Influencers see rank #500, don't share
**Fix:** Make leaderboard visible, send notifications, blue badge

### #6: Contest Rhythm Too Slow (30-40% mid-week churn)
**Failure:** 6 days between engagement moments (Mon draft → Sun end)
**Fix:** Multiple contests (1-week + daily), user in 3-5 simultaneously

### #7: Vague Monetization (5-10 point loss on hackathon)
**Failure:** Judges ask "how do you make money?" and answer is unclear
**Fix:** Document 3-tier pricing, rake model, case studies
